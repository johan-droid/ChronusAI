# 🚀 Deployment Guide

## Vercel (Frontend)

### Quick Deploy

[![Deploy with Vercel](https://vercel.com/button)](https://vercel.com/new/clone?repository-url=https://github.com/yourusername/chronosai)

### Manual Deploy

1. **Install Vercel CLI**
   ```bash
   npm i -g vercel
   ```

2. **Deploy**
   ```bash
   cd frontend
   vercel
   ```

3. **Set Environment Variables**
   ```bash
   vercel env add VITE_API_URL
   # Enter your backend API URL (e.g., https://chronosai-backend.onrender.com/api/v1)
   ```

4. **Deploy to Production**
   ```bash
   vercel --prod
   ```

## Render (Backend)

### Quick Deploy

1. **Connect Repository**
   - Go to [Render Dashboard](https://dashboard.render.com)
   - Click "New +" → "Blueprint"
   - Connect your GitHub repository
   - Render will auto-detect `render.yaml`

2. **Set Environment Variables**
   Required variables (set in Render dashboard):
   - `GEMINI_API_KEY` - Your Google Gemini API key
   - `GOOGLE_CLIENT_ID` - Google OAuth client ID
   - `GOOGLE_CLIENT_SECRET` - Google OAuth secret
   - `GOOGLE_REDIRECT_URI` - https://your-backend.onrender.com/api/v1/auth/google/callback
   - `MICROSOFT_CLIENT_ID` - Microsoft OAuth client ID
   - `MICROSOFT_CLIENT_SECRET` - Microsoft OAuth secret
   - `MICROSOFT_REDIRECT_URI` - https://your-backend.onrender.com/api/v1/auth/outlook/callback
   - `FRONTEND_URL` - Your Vercel frontend URL

3. **Deploy**
   - Click "Apply" to deploy all services
   - Wait for deployment to complete (~5-10 minutes)

### Manual Deploy

1. **Create Web Service**
   ```
   Name: chronosai-backend
   Environment: Python
   Build Command: pip install -r requirements.txt
   Start Command: uvicorn app.main:app --host 0.0.0.0 --port $PORT
   ```

2. **Create PostgreSQL Database**
   ```
   Name: chronosai-db
   Database: scheduler_db
   User: scheduler_user
   ```

3. **Create Redis Instance**
   ```
   Name: chronosai-redis
   Plan: Starter
   ```

4. **Link Services**
   - Add DATABASE_URL from PostgreSQL
   - Add REDIS_URL from Redis
   - Set all environment variables

## Environment Variables

### Frontend (.env)
```bash
VITE_API_URL=https://your-backend.onrender.com/api/v1
```

### Backend (Render)
```bash
APP_ENV=production
SECRET_KEY=<auto-generated>
ENCRYPTION_KEY=<auto-generated>
JWT_SECRET_KEY=<auto-generated>
DATABASE_URL=<from-database>
REDIS_URL=<from-redis>
GEMINI_API_KEY=your-gemini-key
GEMINI_MODEL=gemini-1.5-pro
GOOGLE_CLIENT_ID=your-google-client-id
GOOGLE_CLIENT_SECRET=your-google-secret
GOOGLE_REDIRECT_URI=https://your-backend.onrender.com/api/v1/auth/google/callback
MICROSOFT_CLIENT_ID=your-microsoft-client-id
MICROSOFT_CLIENT_SECRET=your-microsoft-secret
MICROSOFT_REDIRECT_URI=https://your-backend.onrender.com/api/v1/auth/outlook/callback
FRONTEND_URL=https://your-frontend.vercel.app
```

## Post-Deployment

1. **Run Database Migrations**
   ```bash
   # SSH into Render service or use Render Shell
   alembic upgrade head
   ```

2. **Test API**
   ```bash
   curl https://your-backend.onrender.com/health
   ```

3. **Test Frontend**
   - Visit your Vercel URL
   - Try logging in with OAuth

## Monitoring

- **Render**: View logs in Render dashboard
- **Vercel**: View deployment logs in Vercel dashboard
- **API Docs**: https://your-backend.onrender.com/api/docs

## Troubleshooting

### Backend Issues
- Check Render logs for errors
- Verify all environment variables are set
- Ensure database migrations ran successfully

### Frontend Issues
- Check browser console for errors
- Verify VITE_API_URL is correct
- Check CORS settings in backend

### Database Issues
- Verify DATABASE_URL is correct
- Check PostgreSQL logs in Render
- Ensure database is running
